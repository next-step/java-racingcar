## 3단계 자동차 경주 기능 목록
* 설명
  * 초간단 경주 게임 구현
  * 주어진 각 횟수동안 자동차는 '전진/멈춤'을 할 수있다.
* 입력
  * 유저는 몇 대의 자동차로, 몇 번의 이동을 할것인지 입력해야한다.
* 진행
  * 입력받은 자동차 갯수만큼 자동차를 생성한다.
    * 생성한 자동차들은 ArrayList에 담는다. 
    * 최초 시작 시 시작 위치를 지정한다.(init)
  * 입력받은 이동횟수 만큼 반복한다.
  * 전진 조건은 0~9 사이 random 값 출력 후, 만약 출력된 값이 4 이상인 경우에만 가능하다.
    * 4 이상인 경우 현재위치를 이동시킨다.
* 출력
  * 자동차 상태를 화면에 출력한다. 어느 시점에 출력할지에 대한 제약은 없다.


## 4단계 자동차 경주(우승자)
### 설명
* 입력
  * 각 자동차에 이름 부여 (자동차 이름은 5글자 초과 불가) V
  * 자동차 이름은 쉼표(,)를 기준으로 구분 V
* 출력
  * 전진 자동차 출력 시 자동차 이름 같이 출력
  * 게임 완료 후 누가 우승했는지 알려준다. (우승자는 한명이상가능)

### 기능목록
* 입력된 자동차 이름들을 split()한다. V
* Car 클래스에 name 필드 추가 V
  * name 필드에 글자 수 정규식추가 V
* 기록 부분에 자동차 이름도 같이 추가 V
  * 이를 위해 기록방식 변경필요 ProgressRecord 에 name필드 추가 V
* 1등이 누구인지 나아간 숫자로 비교하여 지정
  * GameMove(), GameResult() 중 에서 1등을 가리는 메서드 추가


### 1차 피드백
#### 개선안
* [x] 게임 기록 전체를 저장할 필요없게 그때그때 빼주기
  * 이렇게 되면 불필요한 클래스 선언의 복잡성 줄어듬
  * 그리고 전체 히스토리를 관리하면 100, 1000판 이상하면 데이터 저장은 매우 커짐
    * [x] 불필요한 기록 저장 제거
* [x] Car의 위치를 조정 가능하게 변경
  * 이렇게 해서 누가 1등할지 결과를 예측해 버리기
    * 강의에서 추천받은 방법 : setter VS 생성자 로 우승자 지정

#### 개인목적

  * 즉 여러판 게임에서이 흘럿을 때 특정 자동차는 이동, 특정 자동차는 이동 안하게 하기
* [x] setter 사용 지양
