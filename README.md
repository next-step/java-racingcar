# 자동차 경주 게임
## 진행 방법
* 자동차 경주 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 코드 리뷰 요청을 한다.
* 코드 리뷰 피드백에 대한 개선 작업을 하고 다시 PUSH한다.
* 모든 피드백을 완료하면 다음 단계를 도전하고 앞의 과정을 반복한다.

## 온라인 코드 리뷰 과정
* [텍스트와 이미지로 살펴보는 온라인 코드 리뷰 과정](https://github.com/next-step/nextstep-docs/tree/master/codereview)

## 요구 사항
- 자동차는 이름을 갖는다.
  - 이름은 5자를 초과할 수 없다.
- 자동차는 전진하거나 멈춘다.
  - 랜덤 값이 4 이상이면 자동차는 전진한다.
  - 랜덤 값이 4 미만이면 자동차는 멈춘다.
- 각 회차마다 자동차는 전진하거나 멈춘다.
  - 각 회차 이후 자동차의 위치가 업데이트된다.
- 사용자로부터 자동차의 대수를 입력받는다.
  - `n`대의 자동차를 입력받는다.
  - 유효하지 않은 값(숫자가 아닌 값, 음수)이 입력되는 경우 예외가 발생한다.
- 사용자로부터 시도할 횟수를 입력받는다.
  - `m`번의 시도 횟수를 입력받는다.
  - 유효하지 않은 값(숫자가 아닌 값, 음수)이 입력되는 경우 예외가 발생한다.
- 회차가 끝나면 결과를 출력한다.
  - 각 회차 이후 각 자동차의 위치를 출력한다.
  - 각 회차 결과를 출력할 때에는 자동차의 이름을 함께 출력한다.
- 자동차 경주 게임을 완료한 후 우승자를 출력한다.
  - 우승자는 한 명 이상일 수 있다.

## 모델링
### Car
- `Car`는 위치 정보 `position`을 갖는다.
  - `position`의 기본값은 1이다.
  - `position`은 음수일 수 없다.
- `Car`는 이름 `name`을 갖는다.
  - `name`은 5자를 초과할 수 없다.
- `Car`는 움직일 수 있다.
  - 4 이상의 값을 인자로 받는 경우 전진한다.
  - 4 미만의 값을 인자로 받는 경우 움직이지 않는다.
### Race
- `Race`는 복수 개의 자동차 `cars`를 갖는다.
- `Race`는 시도할 횟수인 `counts`를 갖는다.
- `Race`는 자동차에 랜덤 값 전달을 위한 `RandomGenerator`를 갖는다.
- `Race`는 자동차 경주를 수행한다.
  - 인자로 받은 횟수만큼 자동차 경주를 반복한다.
- `Race`는 우승자를 결정한다.
  - 우승자는 한 명 이상일 수 있다.
