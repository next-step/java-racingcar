### 기능 요구사항

* 자동차의 이름를 입력받는다
    * 자동차 이름은 쉼표(,)를 기준으로 구분한다.
    * 자동차의 이름이 하나라도 입력되지 않는다면 Exception 을 발생시킨다
    * 각 자동차에 이름을 부여할 수 있다. 자동차 이름은 5자를 초과할 수 없다.
    * 자동차의 이름이 5글자차 초과될 경우 Exception 을 발생시킨다.
* 시도할 횟수를 입력받는다
    * 양의 정수 값이 아니라면 Exception 을 발생시킨다
* Random 값을 구한다.
    * Random 값은 0 이상 9 이하여야 한다
* Random 값이 4이상일때 전진한다.
* Random 값이 3 이하일때 멈춘다.
* 자동차 경주를 구현한다.
* 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
    * 형식: pobi : -
* 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다. 우승자는 한명 이상일 수 있다.
    * 형식: pobi, honux가 최종 우승했습니다.
    * 우승자를 출력하지 않으면 Exception 을 발생시킨다
* 핵심 비지니스 로직을 가지는 객체를 domain 패키지, UI 관련한 객체를 view 패키지에 구현한다.
* MVC 패턴 기반으로 리팩토링해 view 패키지의 객체가 domain 패키지 객체에 의존할 수 있지만, domain 패키지의 객체는 view 패키지 객체에 의존하지 않도록 구현한다.
