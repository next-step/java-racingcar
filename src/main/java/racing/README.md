# 기능 목록

1. 사용자 입력
    - 사용자에게 자동차 이름들과 시도할 횟수를 입력받는다
2. 레이스 진행
    - 입력 받은 횟수만큼 반복한다
    - 각 자동차는 한 턴에서 다음 규칙에 따라 이동한다
        - 0-9사이의 랜덤 값을 생성한다
        - 4 이상이면 전진하고 그렇지 않으면 멈춘다
3. 상태 출력
    - 자동차 상태(위치)를 화면에 출력한다
4. 우승자 발표
    - 누가 우승했는지 알려준다
        - 우승자는 한명 이상일 수 있다

## 제약 조건

- 자동차
    - 이름 : 5자 초과 불가능, 쉼표(,)를 기준으로 구분

## 고민

### 빠른 실패가 좋지 않을까?

- 늦은 실패 예시

```readme
경주할 자동차 이름을 입력하세요
ㅔpppppp,pa,a
시도할 회수는 몇 회 인가요?
3
Exception in thread "main" racing.exception.RacingException: 이름은 5자 이하여야 합니다.
	at racing.domain.Car.validate(Car.java:31)
```

- 그렇다면, 차를 만들 때 이름을 검증하는데 차 이름 객체가 따로있는건 어떨까?
    - 장점 : name validation check 하는 로직을 CarName으로 옮길 수 있다

### 역할 / 책임을 나누는 방법?

> 랜덤값 생성 역할 ?

1. 단일 책임 원칙
    - 각 클래스/모듈은 변경해야 할 이유가 하나만 있어야 한다
2. 변경의 이유로 구분
    - 비즈니스 로직 변경
    - UI/입출력 변경
    - 데이터 저장 방식 변경
    - 각각이 다른 클래스에 영향을 주도록
3. 구체적 구분 기준
    - 도메인 로직 : 핵심 비즈니스 규칙
        - Car의 전진 조건 판단-> {Car}
        - Car의 전진 조건 생성 -> {MovementStrategy}
    - 조정/제어 : 흐름 관리
        - 여러 Car를 관리하고 경주 진행 -> {Game}
    - 입출력 : 사용자 / 외부 시스템과의 소통
        - InputView, ResultView
    - 애플리케이션 실행 : 의존성 조립 및 시작
        - GameApplication
